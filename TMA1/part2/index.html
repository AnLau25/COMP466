Unit 1: HTML5 & CSS

This section summarizes HTML5's core features and functionality, describing in detail its structure, elements, and capabilities for web document creation.

Client-side vs Server-side

Client-side programming runs in the user's browser and is responsible for rendering the UI, handling user interactions, and executing scripts without needing to communicate with the server constantly. Examples include HTML, CSS, and JavaScript.

Server-side programming runs on the server and processes client requests, manages databases, performs authentication, and generates dynamic content before sending responses back to the client. Examples include Python, PHP, Node.js, and Java.

A web server holds the data of a domain for users to call via IP and url, it’s main tasks involve:

  • Hosting Websites: Stores the files (HTML, CSS, JavaScript, images, etc.) needed for a website and serves them to users upon request.
  • Handling Requests: Processes it and sends back appropriate responses to requests, like opening the right page for a URL.
  • Running Applications: Web servers can execute server-side code (like PHP, Python, or Node.js) to generate dynamic content.

Basic Structure

All HTML5 follow this structure, regardless of the elements you choose to include.

  • DOCTYPE: Opens your document, declaring it as HTML5.
  • HTML tag: Wraps the entire document as a single structure.
  • Head tag: Contains the metadata of the document (title, styles, scripts and links).
  • Body tag: Contains the visible content of the document, is where you will put what the user will see and interact with.

Introduction to HTML5

HTML5 (HyperText Markup Language) is a markup language used to structure content on the web. HTML5, unlike programming languages such as C++ or Java, it doesn’t include conditional logic or complex algorithms, it’s real purpouse is defining the layout and structure of your webpage, like the skeleton to the human body.

Essential HTML5 Elements

  • Headings: <h1> to <h6> define different levels of headings. This first structure is your go to when generating titles and subtitles with consistent sizing.
  • Paragraphs: <p> defines a paragraph and is your general text structure.
  • Links: <a> href="url">Click here</a> creates a hyperlink, with "click here" being any sort of text input of your choice.
  • Images: <img src="image.jpg" alt="Description"> inserts an image, the alt is also the alternative text that will display if the image is not loaded.
  • Lists: The element that facilitates the creation of esthetically pleasing lists.
    • Unordered List (bullets): <ul><li>Item</li></ul>
    • Ordered List (numbered): <ol><li>Item</li></ol>

Tables

The table structure will allow you to use the space as an Excel sheet. Making divisions for titles and content as needed.

  • thead: Read table head is the element that contains your table title cells.
  • th: Title head.
  • tbody: Read table body, contains the cells that form the body of the table, the data.
  • td: Table data cell.
  • tr: Table rows, which allow you to align your cells in rows for display.

Forms and input types

In HTML5, the <form> element is used to create interactive forms for users to submit data to a web server. Forms are essential for collecting user input, such as text, selections, and files. HTML5 introduced several new form elements and attributes to enhance functionality and usability.

HTML5 clasic structures

  • text: Generic text input field.
  • password: Password input field with hidden characters.
  • button: A button that can execute functions defined by JavaScript.
  • submit: Button that submits the form.
  • reset: Button that resets the form values.
  • radio: Allows to select an option from a group.
  • checkbox: Allows to select one or more options from a group.
  • select: Dropdown to select predefined options.
  • textarea: Multi-line text field.

HTML5 new form structures

  • color: Allows the user to select a color via a pop-up interface.
  • date: Allows the selection of a date in yyyy-mm-dd format.
  • datetime: Allows the user to enter date and time with UTC time zone.
  • datetime-local: Similar to datetime, but without timezone.
  • email: Specialized field for email addresses, with built-in validation.
  • month: Allows to select a month and year in yyyy-mm format.
  • number: Only accepts numeric values and can include minimum and maximum ranges.
  • range: Appears as a slider to select values in a range.
  • search: Designed for entering search queries, visually similar to a text field.
  • tel: Optimized for entering phone numbers, although without built-in validation.
  • time: Allows selection of a time in hh:mm format.
  • url: Validates the entry as a URL.
  • week: Allows the selection of a week and a year in yyyy-Wnn format.

CSS

Cascading Style Sheets (CSS) provide a powerful way to control the appearance and layout of HTML elements. CSS allows for styling elements in various ways, including inline styles, embedded style sheets, and external style sheets.

The CSS laguage renders element following a box model. The box model allows you to control the spacing, borders, and overall layout of elements on a web page, ensuring a more organized and visually appealing design.

Additionally, there are different ways to tell the programs how much space you want an element to take in the document. The most common measurement is pixels (px), but you can also use the typical cm and in, the more relative em (relative to font size) and vw or vh (viewport width and height).

General CSS stylings

  • Absolute Positioning: Removes an element from the normal flow and positions it relative to its nearest positioned ancestor .class {position: absolute;}.
  • Relative Positioning: Positions the element relative to its normal position .class {position: relative;}.
  • Background Image: .class {background-image: url('background.png');}
  • Background Color: .class {background-color: lightgray;}
  • Width and height:Set the dimensions of the element. .class {width: 200px; height: 100px;}
  • Margin: Space outside the border. .class {margin: 15px;}
  • Border: The boundary around the content. .class {border: 5px solid black;}
  • Padding: Space between the content and the border. .class {padding: 10px;}

CSS in HTML

There are three main ways to call/ CSS styles in an HTML document. In order, the closer the style is to its element, the more it affects t. Thus, inline overrides embedded which overrides external styling.

  • Inline Styles: Are directly applied within the HTML element using the style attribute.
    • Useful for applying specific styles to individual elements.
    • Offers the highest level of specificity, overriding embedded and external styles.
    • Best suited for quick, one-off customizations rather than large-scale styling.
  • Embedded Style Sheets: Is defined within a <style> tag located in the <head> section of the HTML document.
    • Provides a centralized way to manage styles for a single page.
    • Allows better organization and readability compared to inline styles.
    • Styles defined here override those in external stylesheets but are overridden by inline styles.
  • External Style Sheets: Is contained in a separate .css file, linked to the HTML document via the <link> element.
    • Promotes reusability and consistency across multiple pages of a website.
    • Eases maintenance by keeping content and design separate.
    • External styles are less specific than embedded and inline styles and can be overridden by them.

Unit 2: JavaScript

JavaScript

JavaScript is programming language widely used for web development, allowing sus to create dynamic and interactive web pages.

Unlike the two previously shown languages, JS does allow us to create variables, loops, functions and all the structures we would usually have access too in object-oriented languages. Much like in those other languages, JavaScript has reserved keywords that are not used but cannot be redefined, including abstract, goto, synchronized, and volatile. Careful when naming your variables.

There are three ways to create variables:

  • var: Function-scoped, hoisted, can be re-declared.
  • let: Block-scoped, hoisted (but not initialized), cannot be re-declared.
  • const: Block-scoped, hoisted (but not initialized), value cannot be reassigned (but objects can be modified).

The data types supported by JavaScript are: Number, String, Boolean, Object, Undefined, Null, Symbol, BigInt

Some of the general operators provided are:

  • Arithmetic Operators: `+`, `-`, `*`, `/`, `%`
  • Relational Operators: `>`, `<`, `>=`, `<=`< /li>
  • Equality Operator: `==`
  • Assignment Operators: `=`, `+=`, `-=`, `*=`, `/=`, etc.
  • Increment & Decrement: `++`, `--`

JS also provides a set of mathematical functions:

  • abs, ceil, floor, round – for rounding numbers
  • sqrt, pow – for square roots and exponents
  • sin, cos, tan – for trigonometric calculations
  • random – for generating random numbers

Calling JS in HTML

While the <script> tag virtually allows you to put scripts anywhere in your code, it is common place to embed the scripts in the head of your document. Still, when handling an important load of scripts, it is a good programming practice to completely separate the HTML and the script by importing the latter via a source link.

JS DOM objects

The document object represents the HTML document currently loaded in the browser and provides methods to dynamically write content, such as document.writeln("Hello, world!");, which moves the cursor to a new line after writing output.

The document object represents the HTML document currently loaded in the browser and provides methods tThe window object, on the other hand, enables JavaScript to interact with users through various methods, including displaying alert boxes with window.alert("This is an alert box!"); and accepting user input through prompt boxes.

The DOM provides various collections::

  • Images Collection
  • Links Collection
  • Forms Collection
  • Anchors Collection

Functions

Functions are a way to efficiently repeat actions or to have certain actions at handy to call without having to re-write them for the sake of having organized code. In js, you create a function by calling “function” giving them a name, (parameters) and a return if necessary.

When creating functions, pay close attention to the scope of your variables:

  • Local scope: Defined within a function, accessible only inside it.
  • Global scope: Defined outside any function, accessible throughout the script.

Arrays and Methods

  • Math Object: Provides mathematical constants and functions.
  • String Object: Series of characters enclosed in quotes.
  • Arrays: Are a collection of elements stored under a single name that can be accessed by using an index starting from zero. Arrays are passed as arguments to functions like other variables.
  • Array Methods:
    • Mutator Methods: Modify the array
    • Accessor Methods: Return new arrays or values without modifying the original array
    • Iteration Methods: Loop through elements
