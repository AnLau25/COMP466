Unit 1: HTML5

This section summarizes HTML5's core features and functionality, describing in detail its structure, elements, and capabilities for web document creation.

Client-side vs Server-side

Client-side programming runs in the user's browser and is responsible for rendering the UI, handling user interactions, and executing scripts without needing to communicate with the server constantly. Examples include HTML, CSS, and JavaScript.

Server-side programming runs on the server and processes client requests, manages databases, performs authentication, and generates dynamic content before sending responses back to the client. Examples include Python, PHP, Node.js, and Java.

A web server holds the data of a domain for users to call via IP and url, it’s main tasks involve:

  • Hosting Websites: Stores the files (HTML, CSS, JavaScript, images, etc.) needed for a website and serves them to users upon request.
  • Handling Requests: Processes it and sends back appropriate responses to requests, like opening the right page for a URL.
  • Running Applications: Web servers can execute server-side code (like PHP, Python, or Node.js) to generate dynamic content.

Introduction to HTML5

HTML5 (HyperText Markup Language) is a markup language used to structure content on the web. HTML5, unlike programming languages such as C++ or Java, it doesn’t include conditional logic or complex algorithms, it’s real purpouse is defining the layout and structure of your webpage, like the skeleton to the human body.

Essential HTML5 Elements

  • Headings: <h1> to <h6> define different levels of headings. This first structure is your go to when generating titles and subtitles with consistent sizing.
  • Paragraphs: <p> defines a paragraph and is your general text structure.
  • Links: <a> href="url">Click here</a> creates a hyperlink, with "click here" being any sort of text input of your choice.
  • Running Applications: Web servers can execute server-side code (like PHP, Python, or Node.js) to generate dynamic content.
  • Images: <img src="image.jpg" alt="Description"> inserts an image, the alt is also the alternative text that will display if the image is not loaded.
  • Running Applications: Web servers can execute server-side code (like PHP, Python, or Node.js) to generate dynamic content.
